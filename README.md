# 這是一個手機網頁+通訊軟體機器人的專案

## 主要功能：記帳機器人

### 功能：
  1. 串接TelegramBot，快捷通過對話的固定格式來新增記帳紀錄
  2. 手機網頁，可查看、編輯、新增、刪除紀錄以及統計數字、圖表
### 風格：
  1. 網頁風格文字緊湊、版型簡單、樣式乾淨簡潔
### 頁面：
  1. 首頁：行事曆+當日紀錄列表
  2. 紀錄編輯頁：可編輯、刪除該筆紀錄
  3. 新增紀錄頁：可新增一筆紀錄
  4. 帳戶頁：可查看、編輯、新增、刪除各帳戶以及其餘額
  5. 統計頁：可查看各分類的統計數字與圖表
  6. 設定頁：可設定分類名稱
### 技術堆疊：
  1. 使用Docker建立伺服器
  2. Database：SQLite
  3. 後端：GoLang
  4. 前端：php
  5. CloudFlared Tunnel 架設於本地伺服器，提供外網存取
### 特殊邏輯：
  1. TelegramBot確認收到格式是否正確、是否可以正確解析，否則回覆正確格式樣式
  2. 網頁首頁上半部分為行事曆，下半部分為選擇當天的紀錄；點擊該筆紀錄即可進入細節編輯，並可刪除該紀錄。
  3. 分類可自定義新增，於網站設定頁中操作
  4. 後端不公開到外網 一律由前端伺服器轉發

### TelegramBot格式：
  1. 新增紀錄
        ```Request
        時間(昨天/今天/明天/日期["2026/01/01"or"01/01"or"1/1"])
        帳戶名稱(可省略，預設為現金)
        收入/支出(可省略，預設為支出) 
        支出金額
        項目名稱
        分類({動態列出分類名稱與編號})
        備註(可省略)
        ```
        ```Response OK
        新增成功！紀錄如下：
        時間：2024-06-01
        帳戶名稱：現金
        類型：支出
        金額：150
        項目名稱：午餐
        分類：飲食
        備註：
        ```
        ```Response Error
        新增失敗！請確認格式是否正確：
        時間(昨天/今天/明天/日期[可接受"2026/01/01"or"01/01"or"1/1"])
        帳戶名稱(可省略，預設為現金{動態列出名稱與編號})
        收入/支出(可省略，預設為支出)
        支出金額
        項目名稱
        分類({動態列出分類名稱與編號})
        備註(可省略)
        ```
  2. 查詢新增格式：
        ```Request
        /新增格式
        ```
        ```Response
        新增紀錄格式如下：
        時間(昨天/今天/明天/日期[可接受"2026/01/01"or"01/01"or"1/1"])
        帳戶名稱(可省略，預設為現金{動態列出名稱與編號})
        收入/支出(可省略，預設為支出)
        支出金額
        項目名稱
        分類({動態列出分類名稱與編號})
        備註(可省略)
        ```
  2. 查詢分類：
        ```Request
        /查詢分類
        ```
        ```Response
        {動態列出分類名稱與編號，以下範例}
        0: 飲食
        1: 交通
        2: 服飾
        3: 3C
        4: 娛樂
        5: 其他
        ```

  3. 查詢帳戶：
        ```Request
        /查詢帳戶
        ```
        ```Response
        {動態列出帳戶名稱與餘額，以下範例}
        現金: 5000
        信用卡: -2000
        銀行帳戶: 15000
        ```
